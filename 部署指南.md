# aiagents-stock 部署方案 (Streamlit + PM2 + Nginx)

本方案专门针对 Streamlit 框架进行优化部署。

## 1. 准备工作

### 1.1 系统依赖安装
```bash
sudo apt update
sudo apt install -y python3-pip python3-venv nodejs npm nginx git vim
# 安装 PM2
sudo npm install -g pm2
```

### 1.2 获取源码
```bash
git clone <你的仓库地址>
cd aiagents-stock
```

---

## 2. 环境配置

### 2.1 创建虚拟环境
```bash
python3 -m venv venv
source venv/bin/activate   #windows下是 venv\Scripts\activate
pip install --upgrade pip
pip install -r requirements.txt
```

### 2.2 配置环境变量
```bash
cp .env.example .env
vim .env # 填入 DEEPSEEK_API_KEY
```

---

## 3. 使用 PM2 启动 Streamlit

推荐直接使用虚拟环境中的 Python 调用 Streamlit 模块启动，这是最稳定且符合标准的方式。

### 3.1 启动命令
在项目根目录下运行以下命令：
```bash
# -- 之后的内容是传递给 Streamlit 的参数
PYTHONUNBUFFERED=1 pm2 start ./venv/bin/python --name "ai-stock" --time -- -m streamlit run app.py --server.port 8503 --server.address 127.0.0.1

```

> [!NOTE]
> **关于端口和地址的说明**：
> - **默认值**: Streamlit 默认端口是 `8501`。如果您不写 `--server.port`，请确保 Nginx 配置中的 `proxy_pass` 也同步改为 `8501`。
> - **8503 端口**: 本项目源码（如 `run.py`）默认推荐使用 `8503`，为了统一，我们在命令中显式指定了它。
> - **地址**: `--server.address 127.0.0.1` 确保服务仅供本地 Nginx 访问（更安全）。

### 3.2 常用管理命令
- **查看状态**: `pm2 status`
- **查看实时日志**: `pm2 logs ai-stock`
- **重启应用**: `pm2 restart ai-stock`

---

## 4. Nginx 反向代理配置

### 4.1 创建站点配置
```bash
sudo vim /etc/nginx/sites-available/ai-stock.conf
```

写入以下内容（注意包含 WebSocket 支持）：
```nginx
# 1. HTTP 自动跳转到 HTTPS
server {
    listen 80;
    server_name alpha.warmmilk.fun;

    # 将所有 HTTP 请求永久重定向到 HTTPS
    return 301 https://$host$request_uri;
}

# 2. HTTPS 配置
server {
    listen 443 ssl;
    server_name alpha.warmmilk.fun;

    # --- SSL 证书配置 ---
    # 请确保这里的路径指向你上传的 .pem 和 .key 文件
    ssl_certificate /etc/nginx/ssl/warmmilk.fun.pem;
    ssl_certificate_key /etc/nginx/ssl/warmmilk.fun.key;

    # 推荐的安全优化配置
    ssl_session_timeout 5m;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:HIGH:!aNULL:!MD5:!RC4:!DHE;
    ssl_prefer_server_ciphers on;

    # --- 原有的反向代理逻辑 ---
    location / {
        # 转发至本地 Python 服务
        proxy_pass http://127.0.0.1:8503;

        # 传递客户端真实信息
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        # 关键：告诉后端现在使用的是 https
        proxy_set_header X-Forwarded-Proto $scheme;                             
                                                                                
        # WebSocket 支持                                                        
        proxy_http_version 1.1;                                                 
        proxy_set_header Upgrade $http_upgrade;                                 
        proxy_set_header Connection "upgrade";                                  
    }                                                                           
}   
```

### 4.2 建立软链接并重启
```bash
sudo ln -s /etc/nginx/sites-available/ai-stock.conf /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

## 5. 注意事项
1. **防火墙**：确保已开放 80, 443 或您自定义的监听端口。
2. **域名解析**：域名必须指向服务器 IP 后才能配置 HTTPS。
