# 智能盯盘使用指南

## 功能定位

智能盯盘负责两件事：

- 基于实时行情和最新策略上下文，给出 `BUY / SELL / HOLD` 决策。
- 当决策是 `BUY` 或 `SELL` 时，生成待人工处理动作，供你手工确认和登记成交。

系统不再自动下单，统一采用提醒与手工登记流程。

## 使用前准备

至少准备以下配置：

```env
DEEPSEEK_API_KEY=your_api_key
DEEPSEEK_MODEL=deepseek-chat
TDX_ENABLED=false
EMAIL_ENABLED=false
WEBHOOK_ENABLED=false
```

如果你需要更稳定的实时行情，可额外配置 TDX：

```env
TDX_ENABLED=true
TDX_BASE_URL=http://127.0.0.1:8181
```

## 基本流程

1. 在资产或盯盘页面创建 AI 盯盘任务。
2. 系统定时拉取行情并调用 AI 进行盘中执行分析。
3. 若结果为 `BUY / SELL`，系统会保存 AI 决策并创建待处理动作。
4. 你根据建议在外部券商终端成交后，再回到系统手工登记买卖。
5. 登记完成后，资产状态、持仓数量、价格预警会自动同步。

## 界面信息说明

- 盯盘列表：展示代码、名称、最近决策、最近状态。
- 决策详情：展示 AI 理由、置信度、关键价位、风险等级。
- 待处理动作：展示待手工执行的买卖建议，处理后会写入成交记录。
- 通知：支持页面提醒、邮件和 Webhook。

## 与价格预警的关系

- AI 盯盘：负责重新分析，产出决策。
- 价格预警：负责监听进场区间、止盈位、止损位是否被触发。
- 二者都只做提醒，不直接下单。

## 常见问题

### 为什么有买入/卖出建议但没有成交？

因为当前模式是手工执行。系统只生成待办和提醒，不会替你下单。

### 为什么非交易时段没有分析？

如果任务开启了“仅交易时段执行”，午休、收盘后和周末会自动跳过。

### 怎样把 AI 建议落到账本？

在待处理动作或持仓页面完成手工登记买入/卖出，系统会同步更新资产和成交记录。
