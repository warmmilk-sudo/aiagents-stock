# 数据源机制完成总结

## 当前结论

系统当前只使用 Tushare 作为 A 股外部数据源。深度分析依赖 Tushare 获取 A 股历史行情；实时行情优先使用 TDX，本地行情服务不可用时由具体模块按配置处理。

## 已覆盖能力

- `data_source_manager.py` 统一管理 Tushare 初始化、重试和错误记录。
- `stock_data.py` 将 Tushare 历史行情标准化为技术指标计算所需的 OHLCV 数据。
- `batch_analysis_service.py` 在深度分析失败时保留底层 Tushare 错误原因，便于定位 Token、权限或接口问题。

## 故障处理

- `token expired`：更新 `TUSHARE_TOKEN` 或检查 `TUSHARE_URL` 代理服务。
- `返回空数据`：检查股票代码、日期范围和 Tushare 接口权限。
- `Tushare 未配置或未初始化`：确认 `.env` 或系统配置中存在有效 Token，并重启后端。

## 不再使用的路径

文档和深度分析链路不再提示其它数据源降级。历史行情失败时应直接暴露 Tushare 原因。
