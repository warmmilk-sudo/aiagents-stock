# 智能盯盘配置说明

## 当前模式

智能盯盘统一运行在“仅提醒”模式：

- AI 负责盘中分析和策略建议。
- 系统负责生成待办、记录决策、发送提醒。
- 最终成交由你在外部交易终端完成，并回到系统手工登记。

## 必填配置

| 配置项 | 说明 | 默认值 |
| --- | --- | --- |
| `DEEPSEEK_API_KEY` | DeepSeek API Key | 无 |
| `DEEPSEEK_MODEL` | 默认模型 | `deepseek-chat` |

最小示例：

```env
DEEPSEEK_API_KEY="your_api_key"
DEEPSEEK_MODEL="deepseek-chat"
```

## 实时行情配置

| 配置项 | 说明 | 默认值 |
| --- | --- | --- |
| `TDX_ENABLED` | 是否启用 TDX 行情源 | `false` |
| `TDX_BASE_URL` | TDX HTTP 服务地址 | `http://127.0.0.1:8181` |

说明：

- 启用 TDX 后，价格预警与部分盯盘分析会优先使用 TDX 实时行情。
- 未启用时会退回默认行情源。

## 通知配置

| 配置项 | 说明 | 默认值 |
| --- | --- | --- |
| `EMAIL_ENABLED` | 是否启用邮件通知 | `false` |
| `SMTP_SERVER` | SMTP 地址 | 无 |
| `SMTP_PORT` | SMTP 端口 | `587` |
| `EMAIL_FROM` | 发件邮箱 | 无 |
| `EMAIL_PASSWORD` | 邮箱授权码 | 无 |
| `EMAIL_TO` | 收件邮箱 | 无 |
| `WEBHOOK_ENABLED` | 是否启用 Webhook | `false` |
| `WEBHOOK_TYPE` | `dingtalk` 或 `feishu` | `dingtalk` |
| `WEBHOOK_URL` | Webhook 地址 | 无 |
| `WEBHOOK_KEYWORD` | 钉钉关键词 | `aiagents通知` |

## 推荐配置模板

### 最小可用

```env
DEEPSEEK_API_KEY="your_api_key"
DEEPSEEK_MODEL="deepseek-chat"
TDX_ENABLED="false"
EMAIL_ENABLED="false"
WEBHOOK_ENABLED="false"
```

### 带实时行情和通知

```env
DEEPSEEK_API_KEY="your_api_key"
DEEPSEEK_MODEL="deepseek-chat"
TDX_ENABLED="true"
TDX_BASE_URL="http://127.0.0.1:8181"
EMAIL_ENABLED="true"
SMTP_SERVER="smtp.qq.com"
SMTP_PORT="587"
EMAIL_FROM="your_email@qq.com"
EMAIL_PASSWORD="your_authorization_code"
EMAIL_TO="target@example.com"
WEBHOOK_ENABLED="true"
WEBHOOK_TYPE="dingtalk"
WEBHOOK_URL="https://oapi.dingtalk.com/robot/send?access_token=xxx"
WEBHOOK_KEYWORD="股票"
```

## 行为说明

- `ai_task.config` 仅保留策略和持仓相关参数，不再包含自动交易字段。
- `price_alert.config` 仅保留进场区间、止盈、止损等提醒参数。
- 历史库中的旧量化字段会在初始化时自动清理。

## 常见问题

### 为什么没有“量化交易配置”页？

因为系统已经移除自动交易链路，只保留分析、提醒、待办和手工登记。

### 价格预警和 AI 盯盘会重复吗？

会有部分重叠，但职责不同：前者做阈值提醒，后者做 AI 决策。
