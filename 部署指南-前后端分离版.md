# aiagents-stock 部署指南（前后端分离）

适用架构：

- 后端：`FastAPI`（入口 `backend.main:app`，默认端口 `8503`）
- 前端：`React 19 + TypeScript + Vite`（开发端口 `5173`）
- 生产：先构建 `frontend/dist`，再由 FastAPI 托管 SPA 静态文件

---

## 1. 快速运行（本机开发）

在项目根目录执行：

```bash
# 1) Python 依赖
python -m venv venv
source venv/bin/activate  # Linux/Mac
.\venv\Scripts\activate  # Windows
pip install --upgrade pip
pip install -r requirements.txt

# 2) 前端依赖
cd frontend
npm install
cd ..

# 3) 环境变量
cp .env.example .env
```

然后开两个终端：

终端 A（后端）：

```bash
source venv/bin/activate
python -m uvicorn backend.main:app --host 127.0.0.1 --port 8503 --reload --no-access-log
```

终端 B（前端）：

```powershell
cd frontend
npm run dev
```

访问：

- 前端：`http://127.0.0.1:5173`
- 健康检查：`http://127.0.0.1:8503/health`

说明：Vite 已配置 `/api` 和 `/health` 反向代理到 `8503`，前端开发时无需额外改地址。

## 2. 环境要求

- Python `3.10+`（建议 `3.11/3.12`）
- Node.js `18+`（建议 `18/20`）
- npm `9+`
- Windows PowerShell 或 Linux Bash

---

## 3. `.env` 最小必填项

从 `.env.example` 复制后，至少确认以下字段：

- `DEEPSEEK_API_KEY`
- `ADMIN_PASSWORD` 或 `ADMIN_PASSWORD_HASH`（二选一）

建议同时确认：

- `LOGIN_MAX_ATTEMPTS`、`LOGIN_LOCKOUT_SECONDS`、`ADMIN_SESSION_TTL_SECONDS`
- `TDX_ENABLED`、`TDX_BASE_URL`（如需启用 TDX）
- `EMAIL_*` / `WEBHOOK_*`（如需通知）

## 4. 生产部署（单进程托管前端）

### 4.1 构建前端

```powershell
cd frontend
npm run build
cd ..
```

执行后应生成目录：`frontend/dist`。

### 4.2 启动后端（托管 API + 前端）

```bash
source venv/bin/activate
python -m uvicorn backend.main:app --host 0.0.0.0 --port 8503 --no-access-log
```

访问：

- `http://<服务器IP>:8503/`（前端）
- `http://<服务器IP>:8503/api/...`（后端 API）

## 5. Linux 持久化运行（systemd）

创建文件：`/etc/systemd/system/aiagents-stock.service`

```ini
[Unit]
Description=aiagents-stock FastAPI
After=network.target

[Service]
User=ubuntu
WorkingDirectory=/opt/aiagents-stock
Environment=PYTHONUNBUFFERED=1
ExecStart=/opt/aiagents-stock/venv/bin/python -m uvicorn backend.main:app --host 0.0.0.0 --port 8503 --no-access-log
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启用与查看：

```bash
sudo systemctl daemon-reload
sudo systemctl enable aiagents-stock
sudo systemctl start aiagents-stock
sudo systemctl status aiagents-stock
journalctl -u aiagents-stock -f
```

## 6. Linux 持久化运行（PM2 推荐）

相比 systemd，PM2 提供更直观的日志管理和自动重启机制。

### 6.1 快速启动

```bash
# 启动后端 (会自动托管 frontend/dist 下的前端文件)
pm2 start "venv/bin/python -m uvicorn backend.main:app --host 0.0.0.0 --port 8503 --no-access-log" --name aiagents-stock
```

### 6.2 使用配置文件（推荐）

在项目根目录创建 `ecosystem.config.js`，然后执行：

```bash
pm2 start ecosystem.config.js
pm2 save # 保存列表，确保服务器重启后自动恢复
```

### 6.3 常用管理命令

- **查看状态**：`pm2 list`
- **实时日志**：`pm2 logs aiagents-stock`
- **重启服务**：`pm2 restart aiagents-stock`
- **停止服务**：`pm2 stop aiagents-stock`

## 9. 常见问题

### 问题 1：访问根路径提示 `frontend dist not found`

原因：未构建前端。

处理：

```powershell
cd frontend
npm run build
cd ..
```

### 问题 2：前端打开了，但请求 API 报错

排查顺序：

1. 后端是否在 `8503` 运行
2. `.env` 是否配置了管理员密码和 API Key
3. 浏览器是否携带了会话 Cookie（登录后）

### 问题 3：日志出现 TDX 连接失败

这是可降级告警，不会阻断系统启动。  
如需启用 TDX，请确认：

- `TDX_ENABLED=true`
- `TDX_BASE_URL` 可访问

### 问题 4：Docker 启动后仍是旧 Streamlit 页面

当前仓库根目录 `Dockerfile` / `docker-compose.yml` 仍偏向旧 Streamlit 启动方式。  
前后端分离架构请优先按本指南手动部署，或将容器启动命令改为 `uvicorn backend.main:app` 并在镜像构建时先执行前端 `npm run build`。
